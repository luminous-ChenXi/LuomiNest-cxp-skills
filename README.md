# LuomiNest CxPlugin Skills

[English](./README.md) | [简体中文](./README.zh-CN.md) | [日本語](./README.ja-JP.md)

LuomiNest skill collection repository — all skills are stored as subdirectories, and the community can contribute new skills via PRs.

## Repository Roles

| Repository | Purpose |
|------------|---------|
| LuomiNest-cxp-registry | Central index registry, maintains `index.json` |
| LuomiNest-cxp-plugin-template | Plugin development template (fork starting point) |
| **LuomiNest-cxp-skills** (this repo) | Skill collection (each skill is a subdirectory) |

Unlike plugins, skills are just Markdown files (`SKILL.md`) or manifests (`manifest.json`) with no complex Python code. Therefore we use a single collection repository instead of one repo per skill.

## Directory Structure

```
LuomiNest-cxp-skills/
├── cooking-assistant/      <- Cooking assistant skill
│   └── manifest.json
├── skill-creator/          <- Skill creator meta-skill
│   └── SKILL.md
├── travel-planner/         <- Travel planning skill
│   └── SKILL.md
├── README.md               <- English (this file)
├── README.zh-CN.md         <- Simplified Chinese
├── README.ja-JP.md         <- Japanese
├── CONTRIBUTING.md
└── LICENSE
```

Each skill subdirectory name = skill id (kebab-case).

## Skill Formats

### SKILL.md (recommended)

YAML frontmatter + Markdown body:

```yaml
---
id: travel-planner
name: Travel Planner
version: 1.0.0
description: Plan travel itineraries based on user needs
author: LuminousCX
license: MIT
tags: [travel, planning, lifestyle]
category: lifestyle
icon: Map
trigger_keywords: [travel, trip, itinerary]
---

# Travel Planner Skill

When users ask travel-related questions...
```

### manifest.json (compatible format)

```json
{
  "id": "cooking-assistant",
  "type": "skill",
  "name": "Cooking Assistant",
  "version": "1.0.0",
  "description": "Recipe suggestions, step-by-step guidance, nutritional analysis",
  "author": "LuminousCX",
  "license": "MIT",
  "icon": "ChefHat",
  "category": "lifestyle",
  "tags": ["community", "free"]
}
```

See full field spec at [docs/development/plugin-system.md](https://github.com/luminous-ChenXi/LuomiNest/blob/main/docs/development/plugin-system.md).

| `cooking-assistant` | Cooking Assistant | Recipe suggestions (based on fridge ingredients), step guidance, nutritional analysis | manifest.json |
| `skill-creator` | Skill Creator | Meta-skill that guides AI to create, modify, validate SKILL.md files | SKILL.md |

## Installation Flow

```
User clicks "Install Skill" in LuomiNest marketplace
  -> Backend downloads the skill subdirectory as zip from this repo
  -> Extracts to local skills/{id}/
  -> cx_skill_loader auto-loads
  -> Skill injected into LLM Prompt
```

Download URL derivation rule (auto-generated by LuomiNest backend):

```
https://github.com/luminous-ChenXi/LuomiNest-cxp-skills/releases/latest/download/skill-{id}.zip
```

Therefore each skill release must create a GitHub Release with a `skill-{id}.zip` asset.

## Contributing Your Skill

See [CONTRIBUTING.md](./CONTRIBUTING.md). Brief flow:

1. Fork this repository
2. Create a new subdirectory `<skill-id>/` in the root
3. Write `SKILL.md` (recommended) or `manifest.json`
4. Submit a PR
5. After merge, maintainers create the corresponding Release and update `LuomiNest-cxp-registry/index.json`
